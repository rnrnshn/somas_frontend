
Insights Module – SOMAS

Design the Insights module for the platform SOMAS – Social Mass Payments & Savings Platform.

The Insights module provides operational analytics and reporting for social transfer programs.

It aggregates data from campaigns, beneficiaries, transactions, and savings programs to provide a clear overview of program performance and financial activity.

The interface must be clean, data-focused, and institutional.

Follow Shadcn Design System principles and maintain a minimal layout with clear visual hierarchy.

Use cards, charts, and tables to present key metrics.

Create the following sections inside the Insights module.

Overview
Financial Insights
Beneficiary Insights
Savings Insights

These sections should be implemented as tabs within the Insights page.

⸻

Overview

Display summary metrics at the top using cards.

Metrics should include:

Total Disbursed Amount
Total Beneficiaries
Active Campaigns
Total Transactions
Total Savings

Include charts showing:

Disbursements Over Time
Transaction Success vs Failure

⸻

Financial Insights

Display financial analytics related to payments.

Charts should include:

Disbursements by Campaign
Disbursements by Province
Transactions by Mobile Money Provider

Include a table showing:

Top Campaigns by Disbursement Amount

⸻

Beneficiary Insights

Display analytics related to beneficiaries.

Charts should include:

Beneficiaries by Province
Beneficiaries by Gender
Beneficiaries by Campaign

Display metrics such as:

Average Payment per Beneficiary
Campaign Participation Rate

⸻

Savings Insights

Display analytics related to savings participation.

Metrics should include:

Total Saved
Number of Savers
Average Savings per Beneficiary

Charts should include:

Savings Growth Over Time
Savings by Campaign
Savings Participation Rate

⸻

Reports and Export

Include an option to export reports.

Export formats should include:

CSV
Excel
PDF

Allow filtering data by:

Campaign
Province
Date Range

⸻

Use UI components such as:

Cards
Charts
Tables
Tabs
Filters

Maintain a minimal and structured interface suitable for operational reporting and donor transparency.
