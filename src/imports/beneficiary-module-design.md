Beneficiary Module – SOMAS

Design the Beneficiary Management module for the platform SOMAS – Social Mass Payments & Savings Platform.

SOMAS is a multi-tenant system used to manage social transfer programs, beneficiaries, financial disbursements, and savings initiatives.

The Beneficiary module is responsible for managing the directory of individuals receiving financial support through campaigns.

The interface must be simple, structured, and institutional. Follow Shadcn Design System principles and maintain a clean, minimal layout.

Create the following screens for the Beneficiary module.

Beneficiaries List

This screen should display a searchable and filterable table of beneficiaries.

Columns should include:

Name
Phone Number
Province
District
Campaign
Total Received
Total Saved
Last Transaction
Status

Include search and filters for campaign, province, and status.

Provide actions such as viewing the beneficiary profile or editing beneficiary information.

Include a button to create a new beneficiary and an option to import beneficiaries via CSV upload.

Create Beneficiary

Design a form for adding a new beneficiary.

Fields should include:

Full Name
Phone Number
Email (optional)
Province
District
Community
Notes

Include validation for phone numbers and required fields.

Beneficiary Profile

Design a detailed beneficiary profile page.

The page should display beneficiary information at the top including name, phone number, location, and status.

Include summary cards showing:

Total Received
Total Saved
Last Transaction Date
Number of Campaigns

The profile page should include tabs.

Overview
Transactions
Savings
Campaigns
Verification

Transactions Tab

Display a table showing the financial transactions associated with the beneficiary.

Columns should include:

Transaction ID
Campaign
Amount
Status
Date

Savings Tab

Display savings participation information including total saved and savings history.

Include a table showing savings contributions.

Campaigns Tab

Display campaigns in which the beneficiary participates.

Include campaign name, program, and participation status.

Verification Tab

Display field verification activity.

Include verification status, enumerator, verification date, and verification notes.

Use components such as tables, cards, tabs, badges, and forms following the Shadcn design system.

The interface should remain minimal, structured, and easy to scan. Avoid visual clutter and maintain a professional institutional tone.
